# 工作排查登记 (Work Tracker Web)

## 最新版本 v2.0

基于聊天内容完整重构的网页版工作排查登记系统，修复了所有已知问题。

---

## ✅ 功能特性

### 1. 用户系统
- **注册 / 登录** — 本地账号，用户名 3-20 位，密码至少 6 位
- **游客模式** — 不保存数据，退出后清空
- **自动登录** — 注册账号下次自动进入
- **退出登录** — 清除当前会话
- **换设备导入** — 登录页提供「导入数据」按钮，导入后自动创建账号

### 2. 任务管理（首页）
- 添加任务：名称 + 类型（每日/每周/紧急）+ 分类 + 标签
- **双击任务名称** 可直接修改
- 筛选栏：全部 / 待办 / 已完成 / 紧急 / 按分类筛选
- 任务操作：更新进度、完成/恢复、删除
- 进度更新模态框：状态（进行中/已完成/受阻）+ 文字记录
- 更新历史显示最近 3 条

### 3. 分类管理
- 6 个默认分类：一般事务、机电工程、设计管理、现场协调、文档审批、会议沟通
- 自定义分类：添加、**修改名称**（双击或编辑按钮）、**删除**（任务自动转回「一般事务」）
- **修改分类后主页自动同步更新**

### 4. 周报页
- 7 天网格视图，当天高亮显示
- 统计面板：总任务、已完成、完成率、待完成、紧急任务
- 待办清单
- 分类统计（带进度条）
- **导出 Excel**（CSV 格式）
- **打印 PDF**（精美排版）

### 5. 提醒系统
- **每日提醒**：早上 9:00 / 下午 17:00
- **每周提醒**：周五 16:00 / 周日 20:00
- **自定义提醒**：任意时间 + 重复（每天/工作日/一次）+ **邮件提醒**
- **提醒页面**：底部导航「提醒」栏目，显示待处理提醒，右上角显示数量角标
- **消除提醒**：进入提醒页面，点击「完成」按钮消除，角标数字自动减少
- **邮件功能**：自定义提醒可勾选「同时发送邮件提醒」，触发时弹出邮件按钮（使用 `mailto:` 链接）

### 6. 数据管理
- **备份数据**：导出 JSON 文件
- **恢复数据**：导入 JSON 文件
- **跨设备迁移**：旧设备导出 → 新设备登录页导入

## 🚀 部署方式

### GitHub Pages（免费）
1. 下载 `work-tracker-web.zip`，解压得到 `index.html`
2. 在 GitHub 创建新仓库（Public）
3. 上传 `index.html` 到仓库
4. 设置 `Settings → Pages → Deploy from branch → main`
5. 访问 `https://你的用户名.github.io/仓库名/`

### Netlify（秒部署）
1. 打开 [netlify.com](https://netlify.com)，用 GitHub 登录
2. 把 `index.html` 直接拖进网页
3. 秒生成公共网址

### 本地打开
直接双击 `index.html` 用浏览器打开即可使用，无需服务器。

## ⚠️ 重要说明

### 跨设备使用
- 网页版数据保存在 **浏览器本地 localStorage**，天然无法跨设备同步
- **换设备时**：旧设备用「设置 → 备份数据」导出 JSON，新设备在登录页点击「导入数据」恢复
- 导入后自动创建同名账号，数据完整还原

### 邮件提醒
- 邮件功能使用 `mailto:` 链接，点击后打开系统邮件客户端
- 需要设备上有邮件客户端（如 Outlook、邮件 App）
- 不支持自动发送邮件（浏览器安全限制）

### 提醒触发
- 提醒检查每 30 秒一次，需要页面保持打开
- 浏览器标签页后台时可能暂停计时，回到前台后自动检查
